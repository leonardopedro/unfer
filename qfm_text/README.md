# qfm_text — QFM-Text

Train the QFM.tex architecture (Tomographic Subspace Recovery) on
token streams, using HRM-Text's data + training process.

> **Honest scope.** This is a quantum-kernel n-gram-family model with
> coherent Krylov smoothing across backoff orders. The success
> criterion is *beating classical interpolated/backoff n-gram
> baselines at equal context order on the same corpus*, and
> demonstrating the QFM pipeline end-to-end at corpus scale (10⁷–10⁸
> tokens, K₂ ~ 10⁵). It will not approach a 1B HRM-Text transformer;
> that comparison is reported for honesty, not as a target.

## Mapping HRM-Text → QFM-Text

| HRM-Text concept | QFM-Text analog |
|---|---|
| token sequence corpus | stream of (context window, next token) pairs |
| embedding + H/L hierarchy | hierarchical dressed vacua: one exact projector per context order o ∈ {1..n} (order-1 ≈ slow/abstract H-module; order-n ≈ fast/detailed L-module) |
| forward pass | encode context → e^{-iHt} → Born-rule sketched probability |
| logits / LM head | per-mode next-token histograms, marginalized under the Born distribution with absolute-discount smoothing to the unigram |
| SGD epochs (epochs=4) | streaming accumulation passes over rotated shard sets (different data per epoch, accumulating counts) |
| FSDP data parallelism | per-shard `ChannelAccumulator`s merged associatively (counts are a monoid) |
| learning rate / EMA | N/A (deterministic counts); smoothing hyperparams (t, λ_o, discount) tuned on validation |
| in-context adaptation | Quantum Bayesian Update (`qfm::bayes`: Likelihood + HMC / `sample_hmc_single`) on the observed prefix |
| checkpoint per epoch | serialized compiled pipeline artifact per pass |
| cross-entropy eval | token-level log-loss / perplexity on the held-out split |

## Architecture adaptations to QFM (all exact projectors)

1. **Hierarchical multi-projector generator.** `H = Σ_o λ_o |0̃_o⟩⟨0̃_o|`,
   one dressed vacuum per context order o over that order's mode
   block. Each term is an exact rank-1 `Operator::ProjectOnto`; the
   sum is Hermitian, rank ≤ n, Krylov dim ≤ n+1. Projectors share
   the Fock vacuum component, so evolution coherently mixes
   orders.
2. **Hashed (collision-accepting) S₂.** `OrderHasher::mode_for`
   assigns `offset_o + splitmix64_seq(ctx_last_o, salt_o) %
   block_size_o`. Fixed memory, no `K2Exhausted`.
3. **Streaming compile.** `QfmPipeline::compile_channels` consumes
   accumulated channel weights directly; the image-decode
   observables are skipped.
4. **Token decode head.** Per-mode next-token histograms
   marginalized under the Born distribution with absolute-discount
   smoothing to the unigram (no zero probability).
5. **Superposition encode.** A query context enters as
   `|Ψ_in⟩ = (1/√n) Σ_o |mode_o(ctx)⟩`.

## End-to-end run

```bash
# Stage 0: download + tokenize WikiText-103 (≈ 200 MB, 16k BPE vocab).
bash qfm_text/scripts/prepare_corpus.sh
# -> $ROOT/hrm_data/wikitext-103-test/{shard_*.bin, manifest.json,
#                                       ATTRIBUTION.txt}

# Stage 1-5: train + eval (see qfm_text/src/bin/*.rs).
cat > /tmp/train.toml <<TOML
manifest_path = "/abs/path/to/hrm_data/wikitext-103-test/manifest.json"
out_dir = "/tmp/qfm_text_out"
epochs = 4
n_orders = 4
block_sizes = [65536, 65536, 65536, 65536]
hist_cap = 64
max_rank = 8
m_shifts = 8
lambda = [1.0, 1.0, 1.0, 1.0]
t = 1.0
discount = 0.75
seed = 0
TOML

cargo run --release -p qfm_text --bin qfm_text_train -- --config /tmp/train.toml
# -> /tmp/qfm_text_out/{checkpoint_epoch{0..3}.qfm, metrics.ndjson}

cargo run --release -p qfm_text --bin qfm_text_eval -- \
  --checkpoint /tmp/qfm_text_out/checkpoint_epoch3.qfm \
  --manifest /abs/path/to/hrm_data/wikitext-103-test/manifest.json \
  --sweep --prompts 5
# -> prints QFM ppl, n-gram ppl, unigram ppl, sweep winner, samples
```

- Tiny synthetic 200-token fixture in `testdata/tiny_fixture.bin`
  (generated by `scripts/generate_fixtures.sh`).

## Crate layout

- `qfm_text/src/corpus.rs` — memory-mapped shard reader + WindowIter
  (Stage 1).
- `qfm_text/src/features.rs` — hashed Level-2 encoder
  (`OrderHasher`, `splitmix64_seq`) (Stage 2).
- `qfm_text/src/accumulate.rs` — `ChannelAccumulator` + `merge` + the
  rayon-parallel `accumulate_shards` (Stage 2).
- `qfm_text/src/model.rs` — `QfmTextModel` (encode_modes, evolve,
  decode_sketched, marginalize, save/load) (Stage 4).
- `qfm_text/src/lm.rs` — `perplexity`, `sample_text`,
  `NgramBaseline` (Stage 5).
- `qfm_text/src/incontext.rs` — Quantum Bayesian Update as
  in-context adaptation (Stage 5).
- `qfm_text/src/bin/qfm_text_train.rs` — train binary (Stage 6).
- `qfm_text/src/bin/qfm_text_eval.rs` — eval binary (Stage 6).
- `qfm_text/scripts/prepare_corpus.py` — WikiText-103 + BPE → shards.
- `qfm_text/scripts/fetch_hrm_text.sh` — pinned HRM-Text clone
  (optional, the Rust crate doesn't depend on it).
- `qfm_text/scripts/generate_fixtures.sh` — deterministic 200-token
  synthetic fixture for tests.
- `qfm_text/testdata/tiny_fixture.bin` — committed test fixture.
- `qfm_text/testdata/README.md` — fixture provenance.
